## 1. Requirements Summary (스펙 요약)

### Goal (스펙 인용)
> "사용자가 태블릿에 그린 박스-화살표 다이어그램 이미지를 Claude Code 세션에 첨부하면, 세션이 IR(JSON)로 인식하고 파이썬 렌더러가 사용자의 기존 PPT 스타일(원본 .pptx에서 추출한 config)을 입혀 **배치까지 완성된, 완전히 편집 가능한 네이티브 도형 .pptx**를 생성한다. 검수·수정은 PowerPoint에서 직접 확인 후 채팅으로 지적하면 IR을 고쳐 재렌더링하는 루프로 처리한다."

파이프라인: `손그림 이미지 →(인식: 세션/VLM) IR JSON →(검수) 확정 IR →(정규화+렌더: 파이썬) .pptx →(PowerPoint 직접 검수) 지적 → IR 수정 → 재렌더`.

### Constraints (스펙 인용, 요약)
- 실행 형태 = **Claude Code 세션**. 인식기 = 세션 자체(VLM), 렌더러 = 세션이 호출하는 파이썬 스크립트. 별도 서비스/CLI 서버/웹 UI 없음.
- 다이어그램 범위 = **박스-화살표 도식**(노드·엣지·중첩 컨테이너·범례).
- 수식·아이콘은 자동 변환하지 않고 **플레이스홀더 박스**로 위치만 표시.
- 스타일 소스 = 사용자 보유 **원본 .pptx 2종**(글꼴만 다르고 색·구조 동일) → 공통 config + 글꼴 스위치.
- 배치는 **손그림 좌표를 존중**(자동 재배치 금지). 정규화는 스냅·정렬·크기통일만.
- 산출물은 덮어쓰지 않고 **증분 파일명(-1, -2, …)** 으로 저장.
- 출력은 **네이티브 pptx 도형만**(이미지 삽입 금지), 커넥터는 도형에 바인딩.

### Non-Goals (스펙 인용)
웹 서비스/제품화, 수식(LaTeX)·아이콘 자동 렌더링, 박스-화살표 외 유형(표·타임라인·플로우차트), 자동 레이아웃(graphviz류), 원샷 100% 자동.

### v1 대비 변경점 (스펙 Technical Context, 스펙 우선)
① 검수 = pptx 직접(오버레이/HTML 원안 폐기) ② 스타일 config = 원본 pptx 추출 + 글꼴 스위치 ③ 레이아웃 정규화 = 품질 1순위 승격 ④ 수식·아이콘 플레이스홀더 추가 ⑤ 실행 형태 = 세션 확정.

## 2. RALPLAN-DR 요약

### Principles (설계 원칙, 스펙에서 도출)
1. **확률적 인식과 결정론적 렌더링의 분리.** 틀릴 수 있는 인식(VLM)과 항상 같아야 하는 렌더링(파이썬)을 **IR(JSON) 계약** 하나로 격리한다. (스펙 Ontology "IR = 검수 대상 / 렌더러 = 결정론적")
2. **사용자 배치 존중.** 손그림 좌표가 진실이다. 정규화는 스냅·정렬·크기통일까지만, 자동 재배치는 금지. (스펙 Constraints, R1 재배치 지옥)
3. **검수 전제 설계.** 원샷 100%는 목표가 아니다. IR이 유일한 수정 지점이고, 재렌더는 값싼 결정론 연산이어야 한다. (스펙 Non-Goals "원샷 100% 자동" 제외)
4. **완전한 편집성 우선.** 산출물은 반드시 네이티브 도형 + 도형에 바인딩된 커넥터. 이미지·프리폼 덩어리 금지. (스펙 Constraints, AC1/AC6)
5. **비파괴 산출.** 어떤 재렌더도 기존 결과를 덮어쓰지 않고 증분 파일명으로 남긴다. (스펙 Constraints, 사용자 전역 규칙)

### 리스크 성격 분류 (게이트 배치의 근거 — v2 신설, v3 정밀화)
- **회복 가능 리스크 = 인식의 "존재/텍스트/방향" 오류(AC3 재현율·precision).** IR 편집 루프가 존재하는 이유가 이런 인식 오류를 값싸게 고치기 위함이다. 누락 노드 추가·오탈자·화살표 방향 반전은 채팅 지적으로 복구된다.
- **부분적으로만 회복 가능한 리스크 = 인식의 "좌표 기하" 오류(v3 신설 구분).** VLM의 bbox 좌표 추정은 인식 파이프라인에서 **가장 약한 고리**다. 40박스 밀집 도식에서 좌표가 틀리면 IR JSON의 bbox를 일일이 수기 이동해야 하므로 "원칙상 복구 가능"과 "AC7(≤10분) 예산 안에서 복구 가능"이 갈린다. 더 중요한 건 **좌표 오차가 클러스터링(B1)의 행/열 군집을 오분류시켜 정규화 자체를 붕괴**시킨다는 점 — 즉 좌표 오차는 회복 가능한 인식 리스크가 아니라 **회복 불가능한 정규화 리스크의 상류 원인**으로 전이한다. 따라서 좌표 기하를 "존재(recall)"와 분리해 별도 계측한다(§3 AC3 좌표 서브게이트, §4 M0).
- **회복 불가능한 존재적 리스크 = 정규화 미감(AC2 재배치 지옥, R1).** 정규화는 렌더러가 원인이므로 IR을 아무리 고쳐도 안 고쳐진다. 사용자가 도구를 버리게 만드는 1순위 치명 실패다.
- **[v5 신설] 제2의 실패 경로 = 밀집 커넥터 라우팅 조악(R13).** 스펙 1순위 AC2는 "박스"를 대상으로 하므로 커넥터 조악은 스펙상 '존재적'까지는 아니지만(라운드 4 Critic도 severity MEDIUM으로 평가), 40박스 밀집에서 바인딩 elbow가 박스를 관통·엉킴으로 자동 재라우팅되면 정렬·크기와 **독립적으로** "다시 그리는 게 낫다"의 트리거가 될 수 있다(라운드 4 Architect·Critic). 따라서 v5는 이를 R1과 분리된 **품질 게이트(R13, §4 M0 elbow 게이트)** 로 못박는다. **[v6 교정 — 라운드 5 MAJOR] 단, 미달 시의 복구책(cxnIdx 정책 고정·M1 "고정좌표 렌더 후 저장 직전 바인딩" 절충)은 '정의된 복구 경로'가 아니라 'M0 마이크로 스파이크로 저장 경로 생존을 실측한 뒤에만 확정되는 후보'다.** 두 복구책 모두 "렌더러가 저장한 초기 경로/연결사이트(cxnIdx)가 PowerPoint 열람 시 라이브 재라우팅을 견디고 살아남는다"는 미검증 전제 위에 있는데, 이는 R13 진단(바인딩 커넥터 경로는 렌더러 XML이 아니라 PowerPoint 자동 재라우팅+cxnIdx가 지배) 자신이 부정하는 바다. 계획은 컨테이너-그룹 바인딩 생존은 M1 스파이크로 실측(§4 M1)하면서 커넥터-경로 생존만 가정하던 비대칭을 안고 있었으므로(라운드 5), **동일한 "미검증 python-pptx 거동 하드커밋 금지" 규율을 R13에도 대칭 적용**한다(§4 M0 2d 스파이크). 생존 실패 시 유일 대안은 비바인딩 커넥터(AC6 위배)이며, **밀집에서 AC6(바인딩)과 AC2(초기 경로 정결)의 충돌이 환원 불가일 수 있음을 정직히 §6 R13에 기록**한다. R1(회복 불가)과 R13(스파이크로 복구 가능성을 실측)의 성격 차이를 명시한다.
- **따라서 게이트 원칙 (v4 핵심 교정 — 라운드 3 반영):** 스펙의 1순위 존재적 기준 AC2(spec line 55: "생성 직후 사용자가 위치를 손봐야 하는 박스 ≤10%")는 정의상 **사용자가 실제로 겪는** 산출물의 기준이고, 사용자는 언제나 **세션 인식 경로**(인식→IR→정규화→렌더)에 있다. 따라서 **AC2 pass/fail은 반드시 세션 인식 경로(생태적으로 타당한 실물)로 판정**한다 — 이것이 M0의 존재적 하드 게이트다. **그러나** 인식 경로만 보면 "못생김의 원인이 R1(정규화)인가 R10(인식 좌표)인가 role 오분류인가"를 지목하지 못하므로, v3의 **골든/좌표/role 분해는 폐기하지 않고 '귀속(attribution) 레이어'로 재배치**한다. 즉 골든 IR 경로·좌표 서브게이트·role 서브게이트는 그 자체가 pass/fail이 **아니라**, 인식 경로 AC2가 fail일 때 **수정 방향을 라우팅**하는 진단 도구다: 골든 경로도 fail이면 정규화(B1) 자체가 깨진 것(→ 임계 재조율 또는 B3 승격), 골든은 GO인데 인식이 fail이면 좌표/role 인식 문제(→ 인식 개선). 이 라우팅 로직은 이미 v3 계획에 존재하므로(§4 NO-GO, §7) **게이트 '밖 배제'를 게이트 '해석'으로 재사용할 뿐 추가 재료 0**이다. (v3가 인식 경로를 '참고 지표(게이트 아님)'로 강등해 반사실 위에서 성공을 측정한 라운드 3 지적을 정면 수용 — §9 Changelog v4 참조)
- **재결합 비용의 정직한 수용 (라운드 3 개선 반영):** 인식 경로 AC2를 게이트로 삼으면 **R1 검증이 인식 성숙도에 재결합**된다 — 인식 좌표/role이 약하면 인식 경로 AC2가 fail하고, R1을 검증하려면 먼저 인식(프롬프트/좌표 힌트/API vision 승격)을 개선해야 하므로 "값싼 M0"의 서사가 **부분적으로 되감긴다**. 이 비용은 회피 불가능한 진짜 대가다. 다만 M0의 핵심 가치는 '값쌈'이 아니라 **회복 불가 R1을 풀 렌더러(M3) 완성 전에 실물 위에서 조기 검증함**이었고, 그 가치는 재결합과 무관하게 보존된다(오히려 인식+정규화의 결합 현실을 M0에서 마주하는 것이 정확하다). 귀속 레이어가 "R1이 아니라 인식 문제"를 지목해주므로 **멀쩡한 정규화 접근을 오손절하지 않는다**. M0에서 인식 개선 서브루프가 필요할 수 있으나 M3 발견 대비 압도적으로 싸다(§8 ADR Consequences에 명시).

### Decision Drivers (상위 3)
1. **재배치 최소화가 진짜 성공 기준.** R1 "재배치 지옥"(정렬·간격·크기가 어색해 전부 다시 배치)이 도구를 버리게 만드는 **치명 실패**로 확정됨 → 레이아웃 정규화 품질이 1순위 드라이버. (AC2가 유일한 "1순위" 태그)
2. **완전한 편집성(네이티브 도형 + 바인딩 커넥터).** 편집 불가 산출물은 요구사항 자체 위배.
3. **최소 인프라 / 저비용 반복 검수.** 세션 실행이라 인프라 0. 검수 루프가 값싸야 전체 소요 ≤10분이 성립.

### Viable Options

#### 결정축 A — 파이프라인 아키텍처
**A1. IR(JSON)을 인식·검수·렌더의 단일 계약으로 두고 렌더러는 순수 결정론 파이썬 (채택)**
- Pros: 인식 오류를 IR 편집으로 국소 수정 가능 / 스타일 변경은 config 교체만으로 재현 / 같은 IR+config → 항상 같은 pptx라 골든 파일 테스트 가능 / 검수 루프가 값쌈.
- Cons: IR 스키마 설계·유지 비용 / 세션이 스키마를 정확히 준수해 JSON을 작성해야 함(검증기로 완화).
- **채택 근거:** Driver 1·2·3을 모두 충족. 스펙 Ontology가 이미 IR을 "core domain, 검수 대상"으로 확정.

**A2. IR 없이 세션이 매 이미지마다 python-pptx 스크립트를 직접 생성·실행 (기각)**
- Pros: 중간표현 불필요 / 초기 구현이 빠름.
- Cons: 매 실행마다 스타일이 흔들려 결정론 상실 / 수정이 코드 재작성이라 검수 루프가 비쌈 / 골든 테스트 불가.
- **기각 근거:** Driver 2(편집성 일관)·Driver 1(재현 안 되면 재배치 지옥 유발) 위배. 스펙/v1이 이미 "LLM 직접 pptx 생성"을 스타일 비일관으로 기각(스파이크 용도만 허용).

#### 결정축 B — 레이아웃 정규화 전략 (Driver 1 직결) — 3안으로 확장(v2)
이 축은 사실 **정규화 공격성 스펙트럼**이다: 아무것도 안 옮김(B2) ↔ 알고리즘이 능동적으로 뭉침(B1) 사이에서, 과교정 위험과 미교정 위험을 교환한다.

**B1. 클러스터링 기반 자동 정규화 (잠정 채택 — v5: M0 A/B로 실증 확정 조건부)**
- 좌표를 행/열로 군집화 → 각 축을 대푯값으로 스냅, 크기는 역할별 중앙값으로 통일, 컨테이너 패딩 균일화. 임계값은 config에 노출하고 **보수적(좁게) 시작** + on/off 플래그.
- Pros: AC2(재배치율 ≤10%)를 직접 겨냥 / 임계값 안쪽만 스냅하므로 "일부러 다르게 그린 것"은 보존.
- Cons: 임계값 튜닝 필요 / **과교정 위험**(의도 변형을 뭉갬) → 테스트·플래그·pre-mortem으로 방어.
- **[v5 핵심 교정 — 라운드 4 MAJOR] B1 우월성은 "좌표가 중간 정도로만 부정확한 좁은 밴드"에서만 성립한다.** 좌표가 정확하면 B2(스냅만)로 충분해 B1은 과잉설계(YAGNI)이고, 좌표가 부정확하면 B1이 틀린 좌표를 군집화해 붕괴(§2 R10 상류 전이)한다. v4까지 B1 채택은 원리론(B2는 미교정 방치)과 골든(무노이즈) 좌표 위 추정에만 의존했고, **실제 세션 VLM 좌표 위에서 B1이 B2를 이기는지는 검증하지 않았다.** v5는 B1을 **잠정 채택**으로 강등하고, on/off 플래그(§4 M0·M3)를 재사용해 **M0 존재적 게이트에서 동일 세션 인식 IR을 B1(정규화 on)·B2(스냅만)로 각각 렌더해 AC2 재배치율을 나란히 실측**한다. 판정: **B2 대등/우월 → B1 폐기·단순화(YAGNI 실현)**, **[v6 교정 — 라운드 5 MINOR] B1 우월분이 마진 밴드 노이즈(±3%p)를 초과할 때만 B1 채택 확정(실증 근거 확보); 우월하나 ±3%p 이내면 저검정력 승리이므로 YAGNI로 B2 채택**, **둘 다 >13% → 좌표 인식이 근본 병목임을 조기 확정(B3 상시화/전제 재고)**. (N=2·사람 판단의 '명확 우월'은 저검정력이므로 타이뿐 아니라 유의미하지 않은 B1 우월도 단순안 B2로 편향시켜, 노이즈성 2-표본 승리로 불필요한 클러스터링 복잡도를 채택하는 것을 막는다.) 이 A/B는 플래그 재사용이라 추가 재료가 최소다(§4 M0, §7 M0 검증).

**B2. 정규화 최소(그리드 스냅만) (v5: 기각 → M0 A/B 기준선으로 승격)**
- Pros: **과교정 위험 0**. 사용자가 그린 좌표를 사실상 그대로 두므로 R1의 "과교정" 경로에 대해선 오히려 가장 안전하다.
- Cons: 줄맞춤·크기통일이 안 돼 **미교정으로 인한 재배치 지옥**이 남는다. 삐뚤한 줄·들쭉날쭉한 크기를 사용자가 손수 정렬해야 하므로 AC2(1순위) 미달 **가능성**.
- **[v5 재평가] "안전하지만 무능"은 원리론상 참이나 경험적으로 열린 질문이다.** 좌표가 정확+손그림이 잘 그려진 경우 B2가 과교정 0으로 AC2를 통과할 수 있고, 반대로 VLM이 좌표에 노이즈를 더하면 B1이 오히려 악화될 수 있다(라운드 4). 따라서 v5는 B2를 계획시점 하드 기각하지 않고 **M0 A/B의 대조군(기준선)** 으로 승격한다. B1이 B2를 실측으로 이기지 못하면 단순한 B2를 채택한다(YAGNI).

**B3. 반자동 — 클러스터링 + 사용자 정렬 힌트 (v5: M2 옵션으로 전진)**
- IR에 사용자가 정렬 그룹/기준선을 힌트로 명시하면(예: `align_hint: {row: [n1,n2,n3]}`) 그 그룹만 스냅, 나머지는 좌표 보존.
- Pros: 과교정·미교정을 사용자 의도로 직접 통제 → R1에 가장 강함. **[v5] 검수 루프에서 "이 셋을 한 줄로"식 채팅 지적으로 위치를 값싸게 교정하는 유일 채널** — R1이 마진 밴드 경계(7~13%)에서 잔존해도 사용자가 PowerPoint 수동 정렬(AC7 위협) 대신 채팅으로 고칠 수 있다.
- Cons: 힌트 작성 부담. **단, 이는 자동 배치를 대체하는 게 아니라 잔여 오차를 교정하는 보조 채널**이다 — B3가 전면 필요(사용자가 배치를 다시 지정)해지면 그것은 도구의 핵심 가치(자동 배치) 실패 신호이므로(라운드 4 Architect), B3는 **잔여 교정용**으로만 위치시킨다.
- **[v5 위상 변경 — 라운드 4 MINOR] M3 폴백 → M2 옵션.** 골든 IR 라벨링에 `intent_align` 스키마 형태가 **이미 존재**(§4 M0 골든 라벨)하므로 이를 런타임 입력으로 승격하는 비용이 작다. AC2가 밴드 경계에서 GO돼도 사용자가 매 산출물 4~5박스를 PowerPoint 수동으로 고치는 AC7 위협을 방어하기 위해 **가장 값어치 있는 위치-수정 채널을 검수 루프(M2)에 확보**한다. B1이 M0 A/B에서 붕괴(둘 다 >13%)하면 B3는 폴백에서 **상시 채널**로 재승격.

> 참고: 스펙에서 이미 기각된 대안(graphviz 자동레이아웃, SVG→PPT 경유, 이미지 출력, OpenCV 규칙기반 인식, 검수용 오버레이/HTML)은 재론하지 않는다.

**목적(v2 확장, v4 재구성):** v1의 "순수 IR 인식 스파이크"를 **end-to-end 얇은 슬라이스**로 확장한다. 회복 불가능한 R1(정규화 미감)을 6~8일 뒤(M3)가 아니라 **1~3일 안에 실제 pptx 위에서** 사람이 "이걸 다시 만들 것인가?"로 판정한다. **[v4 교정] R1 pass/fail은 사용자가 실제 겪는 세션 인식 경로로 판정**한다(§2·§3 AC2) — 골든 경로로만 판정하면 사용자가 결코 살지 않는 반사실 위에서 성공을 재게 되므로(라운드 3). **골든 IR 경로·좌표·role 서브게이트는 pass/fail이 아니라 인식 경로 AC2가 fail일 때 "원인을 정규화 vs 인식으로 라우팅"하는 귀속 레이어**로 둔다. 회복 가능한 인식 리스크(AC3 존재·좌표·role)는 같은 M0에서 **각각 성격에 맞는 별도 지표**로 동시에 계측한다.

**[v6 신설 — M0 게이트 순서 M0a → M0b, 라운드 5 MAJOR] 값싼 조기 손절 규율 복원.** v5의 M0는 회복 불가 R1을 실물로 조기 검증한다는 존재 근거를 지녔으나, 인식 충실도 프로브를 recognition_eval 7지표 빌드·합성 스윕·스텁 elbow 렌더러·B1/B2 A/B·elbow 게이트와 한 덩어리로 융합해, 정작 최상류·최불확실·가장 값싸게 테스트 가능한 하중 전제(§2 R10 "가장 약한 고리": 세션 VLM이 쓸 만한 bbox 좌표를 산출하는가)를 값비싼 하류 건설과 재결합했다. 계획의 정교한 하부 구조 전체(IR 스키마·B1 클러스터링·좌표/role 서브게이트·합성 스윕·A/B·귀속 레이어)는 이 단일 미검증 전제 위의 하류 기계장치이므로, 합리적 순서는 그 전제를 홀로·먼저 게이트하는 것이다. v6는 M0를 두 서브게이트로 재배열한다(§7 검증 순서가 이미 인식 계측을 렌더 앞에 두므로 재료 추가 거의 0이고 현행 M0를 strictly dominate):
- **M0a (반나절, 벌거벗은 인식-좌표 프로브 — 하중 전제 단독·선행 게이트):** 렌더러·정규화·A/B·elbow 게이트·`intent_align`/`size_class` 라벨 **없이**, 실제 예시 손그림을 세션이 인식→bbox 하고, **최소 골든**(bbox + 노드/엣지 존재 + role만)과 centroid·IoU·recall을 **일회용 계측**(노트북 수준 스크립트, 완성 `recognition_eval.py`가 아님)으로 비교한다. **좌표 충실도가 가망 없고 프롬프트 격자·상대위치 힌트 + API vision 승격으로도 못 살리면 여기서 손절 — 비용 반나절.** (스텁 렌더러·정규화·A/B를 짓기 전에 손절하므로 손실 최소. 이 세계에서 6~8일 계획 전체가 R1 실패로 귀결하는 것을 반나절 프로브로 조기 차단.)
- **M0b (M0a 통과 시에만 진입 — 현행 M0의 나머지 전부):** `recognition_eval.py` 완성(7지표)·`intent_align`/`size_class` 전체 골든 라벨·정규화 프로토타입·스텁 elbow 렌더러·B1/B2 A/B·elbow 라우팅 게이트(R13)·존재적 AC2 게이트를 짓는다. 값비싼 하류 건설은 하중 전제(인식 충실도)가 M0a에서 이미 탈리스크화됐으므로 정당하다.

이 분할은 v5가 "값싼 M0 부분 되감김"(§8 Consequences)으로 **수용**하던 리스크-우선순위 역전을, 되감김의 **최소화**로 전환한다(라운드 5 improvement). 아래 렌더러 최소판·게이트·생성파일·검증 절차는 별도 표기가 없으면 M0b에 속한다.

**M0에 포함하는 렌더러 최소판(스텁) — v3: 저비용·고레버리지 피델리티만 선별 전진:** 네이티브 사각형 + **elbow(직교) 라우팅 커넥터** + 정규화 프로토타입(**role별 크기통일 포함**). 근거(라운드 2): AC2("어색함→재배치 충동")는 레이아웃 지각과 얽히는데, **직선 대각 커넥터는 그리드에 스냅된 박스도 어긋나 보이게 만들고, 크기 들쭉날쭉도 지각을 흔든다** — 이 지배적 시각 교란을 M0에서 제거해야 AC2 신호가 R1을 순수하게 가리킨다. 반대로 **값비싼 정밀 스타일(테마색 schemeClr 해석·latin/ea/cs XML·정밀 RGB 대조)·화살표 머리 정밀화·플레이스홀더·글꼴 스위치는 M1/M3에 그대로** 둔다. elbow는 계획 스스로 R4에서 "알려진 lxml 패턴"이라 명시했고, role 크기통일은 이미 M0 정규화 프로토타입에 있으므로(B1) 렌더러가 그 크기를 반영만 하면 돼 추가 비용이 작다 → M0 예산 안에 흡수.

**[v4 실측 → v5 명시 게이트로 승격] elbow 라우팅 게이트 (R13, 라운드 4 MAJOR):** elbow를 M0로 당기는 유일 명분은 "직선 대각 커넥터 confound 제거"인데, `stCxn`/`endCxn`로 바인딩된 bent 커넥터의 **실제 경로는 렌더러 XML이 아니라 PowerPoint의 자동 재라우팅 + 연결 사이트(cxnIdx) 선택이 지배**한다. 40박스 밀집 도식에서 바인딩된 elbow가 박스를 관통·엉킴으로 재라우팅되면, elbow가 confound를 **제거**하는 게 아니라 **다른 confound를 추가**해 AC2 판정을 오염시키고, 나아가 정렬·크기와 독립적으로 "다시 그리는 게 낫다"의 트리거가 될 수 있다(라운드 3~4). v4는 이를 '실측·격리'만 했으나 pass/fail 임계가 없어 밀집 라우팅이 나빠도 M1 GO가 날 수 있었다. **v5는 이를 명시 게이트로 승격**한다:
- **게이트 임계(초기값, M0에서 재확정):** 밀집 골든 케이스(≈40박스)에서 바인딩 elbow의 **박스 관통 커넥터 수 ≤ 전체 커넥터의 10% AND 과다 교차(동일 커넥터가 3회 이상 타 도형·커넥터와 교차) ≤ 15%**. **[v6 정의 — 라운드 5 MINOR] '관통'의 기하 정의: 커넥터 폴리라인 세그먼트가 자신의 끝점(stCxn/endCxn 대상)이 아닌 노드의 bbox 내부와 교차하는 경우** (bbox 경계만 스치는 것은 제외, 내부 침범만 카운트) — '과다 교차'가 3회로 정의됐듯 '관통'도 재현 가능하게 못박아 두 개발자가 다르게 계측하지 않게 한다. cxnIdx 선택 정책(어느 연결 사이트를 고를지)도 함께 기록.
- **[v6 교정 — 라운드 5 MAJOR] 미달 시 복구 후보(프로젝트 손절 아님, 단 '정의된 복구 경로'가 아니라 스파이크로 생존을 실측한 뒤에만 확정):** 후보 (a) **cxnIdx 선택 정책 고정**(예: 방향별 최근접 사이트 강제)해 재측정, (b) **M1 절충 "초기 렌더는 고정좌표 경로 + 저장 직전 stCxn/endCxn 바인딩 부여"**. **두 후보 모두 "렌더러가 저장한 초기 경로/cxnIdx가 PowerPoint 열람 시 라이브 재라우팅을 견디고 살아남는다"는 미검증 전제 위에 있다** — 이는 위 게이트 진단(바인딩 경로는 렌더러 XML이 아니라 PowerPoint 자동 재라우팅+cxnIdx 지배) 자신이 부정한다. 따라서 **M0 2d에 마이크로 스파이크를 못박는다**: *밀집 바인딩 elbow pptx를 렌더 → PowerPoint에서 열어 → 아무것도 안 옮긴 상태에서 저장 경로가 재라우팅되는가·초기 경로가 생존하는가·cxnIdx가 재선택되는가를 실측*하고 `_thinking/analysis/00X-m0-slice.md`에 기록. **생존이 실측된 경우에만** 해당 후보를 복구 경로로 확정. **생존 실패 시 유일 대안은 비바인딩 커넥터(AC6 위배)이며, 밀집에서 AC6·AC2 충돌이 환원 불가일 수 있음을 §6 R13에 정직히 기록**(컨테이너-그룹 바인딩 생존을 M1 스파이크로 실측하면서 커넥터-경로 생존만 가정하던 비대칭을 R13에도 동일 규율로 대칭 적용). 확정 시 절충은 M1 스코프에 옵션으로 명시(§4 M1).
- AC2 설문(3) 카운트 제외는 유지하되(측정 순수성), **제외된 항목이 이 게이트로 반드시 판정**되게 한다. 결과를 `_thinking/analysis/00X-m0-slice.md`에 기록.

**[v5 신설 — M0 착수 전 결정] IR 좌표계 확정 (라운드 4 MAJOR, R10 스키마 선제 방어):** IR bbox 좌표 표현(절대 픽셀 vs 정규화)은 Principle 1(IR = 단일 계약)의 핵심 계약 결정이자 최상류 병목 R10(좌표 기하 오차)에 직결되는데 v4까지 미결정이었다(v1 초안은 canvas {1280,720} 절대px, 그러나 AC3는 정규화 % 측정 — 표현·측정 불일치). VLM은 고해상도 절대 픽셀 좌표 추정에 구조적으로 약하고 태블릿 사진은 해상도가 매번 다르므로, **[v6 교정 — 라운드 5 MAJOR] v6는 좌표계를 등방(isotropic) 정규화 정수로 확정**한다(v5의 축별 독립 0~1000이 비정방 캔버스에서 1‰(x)≠1‰(y) 이방성 구멍을 남긴 것을 교정, §3 AC3): `canvas`는 종횡비(예: 16:9)만 보유하고, **장축을 0~1000, 단축을 0~round(1000×단축/장축)** 로 정규화(16:9면 x∈0~1000, y∈0~562), 모든 `bbox`의 x·y·w·h는 이 등방 단위 정수. 1‰가 두 축에서 동일 물리 길이를 가지므로 centroid 2D 거리·B1 클러스터링 거리·"박스폭 X%" 임계가 단일 척도에서 돈다. `ir/schema.md`·`ir/schema.json`에 이를 명시하고 recognition_eval의 오차 단위(‰)와 일치시킨다. 이 결정은 R10을 프롬프트 튜닝뿐 아니라 **스키마 설계로 선제 방어**한다(§9 Q7에서 최종 확인, 사용자가 절대 px 유지를 원하면 재고).

**[v5 신설 — M0 스코프 경계 정직 표기] (라운드 4 improvement 수용):** v5가 M0에 더한 것(B1 vs B2 A/B·elbow 라우팅 게이트)은 대부분 **새 빌드가 아니라 이미 생산되는 산출물 위의 판정 기준**이다 — A/B는 on/off 플래그 재사용(2벌 렌더), elbow 게이트는 이미 실측하던 데이터에 임계 부여, 좌표계는 M0 착수 전 1회 결정. 따라서 M0 소요는 2~3일을 유지한다. **역배치 방지 원칙:** role confusion 서브게이트는 M0에 존치(크기통일이 role의 하류라 정규화 판정 전에 봐야 하므로 M1 이동 불가), 대신 **elbow의 정밀 폴리시(화살표 머리·엣지케이스 라우팅)는 M3에 존치**하고 M0는 '박스 관통 임계 게이트'만 본다. 이로써 "얇은 수직 슬라이스" 정체성을 판정-비용 위주로 보존한다.

- `docs/golden-ir-labeling.md` (v3 신설, v4 보강) — **골든 IR 라벨링 프로토콜**: 누가·어떤 도구로 bbox를 찍는지, 좌표 정밀도 허용치(캔버스 %), intent_align/size_class/**role 정답** 부여 기준. 이 골든 IR이 recall/precision·좌표오차·의도정렬·**role 정답**의 공동 담체이므로 라벨 품질이 M0 게이트 전체 신뢰도를 좌우함을 명시. **[v4 → v5 강화] 라벨러 간 일관성 검증**: 개인 도구라 '2인 교차 라벨'은 비현실적이므로(라운드 4 improvement), **재현성 스팟체크를 '또는'이 아닌 필수 절차로 못박는다**: 동일 손그림을 시차를 두고 재라벨해 편차를 측정하고 허용치를 수치화 — **bbox centroid 재라벨 편차 중앙값 ≤ 좌표 서브게이트 X의 절반, role·intent_align 라벨 불일치 ≤ 5%**. 초과 시 라벨 프로토콜을 조이고 재라벨. **[v5 신설] AC2 판정자 캘리브레이션**: 같은 스텁 pptx를 시차를 두고 2회 판정해 "위치 손본 박스 수" 카운트 편차를 기록(정렬 지각이 색 그룹핑·테두리 등 스타일에 부분 의존하므로 무스타일 스텁 위 판정의 신뢰도를 계측) — 편차가 밴드 폭(±3%p)을 넘으면 판정 기준을 문서화해 재판정. 라벨 편향이 M0 게이트를 조용히 오염시키지 않게 한다(R11).
- `eval/recognition_eval.py` (v4 확장) — 세션 IR vs 골든 IR로 노드/엣지 **재현율·precision·F1 + bbox centroid 오차(중앙값·p90)·IoU(중앙값·IoU<0.5 노드 수) + role 정확도(confusion)** 산출(AC3 존재·좌표·role 서브게이트). **골든↔인식 IR diff 크기(수정 필요 노드/엣지 수) 출력**(AC7 조기추정용). **[v4 신설] 합성 노이즈 스윕 유틸**: 골든 좌표에 점증 노이즈를 주입해 정규화가 행/열을 오분류하기 시작하는 오차 크기를 측정 → 좌표 서브게이트 임계 X·Y를 held-out 원리로 도출(§3 AC3, 순환 회피).
- `normalize/layout.py` (프로토타입) — 클러스터링 스냅/정렬/**role별 크기통일** 최소판 + **의도-기반 과교정/미교정 카운터**(golden IR의 intent_align/size_class 대비 산출, AC2 비순환·의도-인지 지표). **[v5] 정규화 on/off 플래그를 A/B 스위치로 노출** — on=B1(클러스터링), off=B2(스냅만) — 동일 세션 인식 IR을 두 벌 렌더해 M0 존재적 게이트에서 재배치율 대조(B1 채택의 실증 근거).
- `render/shapes.py`, `render/connectors.py`, `render/renderer.py` (스텁) — 사각형 + **elbow 라우팅 커넥터** + role 크기통일 반영으로 IR→pptx 렌더(골든 경로·인식 경로 둘 다).
- `cli.py` (최소) — `render <ir.json> -o outputs/<name>.pptx`.
- `_thinking/analysis/00X-m0-slice.md` — 결과 기록(append-only, 번호는 기존 최댓값+1). **골든 IR 라벨 품질·좌표 정밀도·AC7 diff 추정치 포함.**

**소요(v6 분리 재추정):** **M0a 반나절 + M0b 2~2.5일 = 총 약 2.5~3일.** M0a(벌거벗은 인식-좌표 프로브)는 렌더러·A/B·게이트 없이 일회용 계측만이라 반나절이고, 손절되면 여기서 종료(하중 전제 탈리스크 실패 시 손실 최소). M0b는 M0a 통과 시에만 recognition_eval 완성·스텁 elbow 렌더러·B1/B2 A/B·elbow 게이트·존재적 AC2 게이트를 짓는다(elbow는 알려진 lxml 패턴, role 크기통일은 정규화에 이미 존재). 이 분할은 v5의 "값싼 M0 부분 되감김"을 **되감김의 최소화**로 전환한다(§8 Consequences). 회복 불가 R1을 M3(6~8일 뒤)가 아니라 여기서 실물로 조기 검증하는 가치는 보존한다.

**게이트 구조 (v4 재구성 — 존재적 pass/fail 1 + 귀속 서브게이트):**
- **[존재적 게이트 — pass/fail을 정함] 인식 경로 AC2 + B1 vs B2 A/B:** 세션 인식 IR → 정규화 → 스텁 렌더(elbow·role 크기통일)로 pptx를 **표본 전부** 뽑되 **정규화 on(B1)·off≈스냅만(B2) 두 벌**을 낸다. PowerPoint에서 **위치 손본 박스 수 / 전체**를 마진 밴드로 판정(**≤7% GO / 7~13% 표본 추가 / >13% NO-GO**). 설문 (1) 위치·정렬·크기 틀림만 카운트, (2) 스타일 조악·(3) 커넥터 경로 조악은 제외. **[v5/v6] A/B 판정: B2가 대등/우월이면 B1 폐기·B2 채택(YAGNI), [v6] B1 우월분이 밴드 노이즈(±3%p)를 초과할 때만 B1 확정(§8 ADR에 실증 연결)·±3%p 이내면 YAGNI로 B2, 둘 다 >13%면 좌표 인식 근본 병목 조기 확정 → B3 상시화/전제 재고.** **[v6 다표본 집계] 종합 GO/NO-GO는 최악치: 어느 표본이라도 >13%면 NO-GO, 모든 표본 ≤7%면 GO, 그 외 표본 추가 후 재판정(§3 AC2).** — **이 게이트(채택 정규화의 인식 경로 AC2)만이 M1 GO/NO-GO를 정한다.**
- **[품질 게이트 — pass/fail, R13] 밀집 elbow 라우팅:** 바인딩 elbow의 박스 관통 커넥터 ≤ 전체 10% AND 과다 교차 ≤ 15%(초기 임계). 미달 시 프로젝트 손절이 아니라 cxnIdx 정책 고정(M0) 또는 M1 절충(고정좌표+저장전 바인딩) 발동. AC2 설문(3)에서 제외한 항목의 책임 게이트.
- **[귀속 서브게이트 — pass/fail 아님, fail 원인 라우팅용] 다음 넷:**
  - (a) **인식·존재(R2/R3):** 노드·엣지 재현율 각 ≥ 80% AND precision ≥ 0.85(환각노드 ≤ 골든의 10%). — AC3.
  - (b) **인식·좌표 기하(R10 상류 병목):** 중앙값 centroid 오차 ≤ 박스폭 X% AND 중앙값 IoU ≥ 0.7 **AND [v4 꼬리] p90 centroid ≤ Y% AND IoU<0.5 노드 ≤ 전체 10%**. X·Y는 합성 노이즈 스윕으로 held-out 도출. — AC3 좌표.
  - (c) **인식·role(v4 신설):** role confusion ≤ 임계(초기 5~10%). 크기통일이 role 하류이므로 정규화 판정 전에 본다. — AC3 role.
  - (d) **골든 경로 AC2(귀속 기준선):** 골든 IR → 정규화 → 렌더 pptx의 위치 손본 박스 ≤ 10%. 골든도 fail이면 정규화(B1) 자체 결함 신호(→ B3 승격), 골든 GO면 인식 경로 fail은 (b)/(c) 인식 문제로 귀속.
  - (e) **기계 백업(의도-기반 비순환):** 의도 미교정 + 과교정 ≤ 전체 10%. — AC2 기계 백업.

**판정 로직(v4):** 인식 경로 AC2가 GO면 → M1 GO(서브게이트가 초록이면 견고, 붉으면 잠재 부채로 기록). 인식 경로 AC2가 NO-GO면 → 서브게이트로 원인을 라우팅(아래 NO-GO 경로).

**NO-GO 복구/라우팅 경로 (v4 — 인식 경로 fail을 귀속 레이어로 진단):**
- **인식 경로 AC2 fail + 골든 경로 AC2도 fail:** 정규화 자체 결함. **[v5] 먼저 B1 vs B2 A/B 결과를 본다** — B2(스냅만)가 B1보다 나으면 B1을 폐기하고 B2로 단순화(과교정이 원인이었음). 둘 다 나쁘면 의도 과교정↑이면 임계 좁히고 미교정↑이면 넓혀 **다른 표본으로 재측정**(단일표본 노이즈 배제) → 그래도 B1·B2 둘 다 두 카운트를 10% 이하로 못 내리면 **좌표 인식이 근본 병목**이므로 **B3(사용자 정렬 힌트) 상시화** 검토(false-NO-GO로 곧장 B3 직행 금지).
- **인식 경로 AC2 fail + 골든 GO + 좌표 서브게이트(b) fail:** 좌표 인식 문제. 인식 프롬프트에 좌표 격자 힌트·상대 위치 서술 보강 후 재시도 → 그래도 미달이면 **API vision 인식 승격** 검토. (정규화는 손대지 않음.)
- **인식 경로 AC2 fail + 골든 GO + role 서브게이트(c) fail:** role 인식 문제. 프롬프트에 role 판별 힌트 보강·용어사전 강화 후 재시도 → 미달이면 API vision 승격. (정규화 손대지 않음.)
- **[v4 신설 — 잔여 꼬리 케이스] 인식 경로 AC2 fail + 골든 GO + 좌표·role 서브게이트 모두 GO:** 서브게이트가 다 통과하는데도 합성 AC2가 나쁜 경우 = 중앙값 서브게이트가 상계 못한 fat-tail이거나 미모델링 실패 모드. **먼저 꼬리 지표(p90·IoU<0.5)를 재확인** — 꼬리 지표도 GO였다면 임계 X/Y가 느슨한 것이므로 합성 노이즈 스윕으로 재도출 후 재판정, 그래도 잔존하면 표본을 늘려 실패 모드를 특정한다(오손절 방지).
- **존재적 게이트 착수 전 손절:** 렌더러 미완 상태이므로 어느 경로로도 복구 불가 판정 시 손절 손실이 최소(스텁만 폐기).

**[v4 신설] AC2 × AC7 조인트 텐션 해소 (인식 경로 게이트의 부수 효과):** 인식 좌표가 약하면(§2 "가장 약한 고리", R10 높음/높음) 인식 경로 AC2를 달성하려 40개 bbox를 수기 이동해야 하고, 이는 AC7(≤10분)을 깬다 — 두 1순위/부차 기준이 인식 경로에서 충돌할 수 있다. M0에서 이 조인트 텐션의 해소 예산을 못박는다: (1) **좌표 힌트 프롬프트의 실효를 M0에서 실측**(격자·상대위치 서술이 centroid 오차를 유의미하게 줄이는가 — recognition_eval로 before/after 비교), (2) 실효가 부족하면 **API vision 인식 승격을 예산 내 옵션으로 승인**(세션 VLM 대비 좌표 정밀도 개선분 측정). 둘 다 실패하면 AC2와 AC7은 현 인식 성숙도에서 동시 충족 불가임을 정직히 기록하고 B3(사용자 정렬 힌트, 좌표 수기이동 대체) 승격으로 AC7을 지킨다.

### M1 — 렌더러 고도화 (스텁 → 완전 충실도 결정론 렌더러)
**목적(v2 변경):** M0에서 접근법이 GO된 뒤, 스텁 렌더러를 **완전한 스타일 충실도**로 끌어올린다. 스펙 근거: Goal "결정론 렌더러", Constraints "네이티브 도형만·커넥터 바인딩", 변경점 ②.

**M1 착수 전 명시 결정 (v2 신설, v3 수정 — 하드커밋 → 기본값 + 스파이크 검증):**
- **컨테이너 기본 = 평면 배경 사각형(비그룹, z-order 뒤).** M0/M1의 결정론 게이트(AC1 카운팅·AC6 바인딩)는 이 flat 기본 경로로 판정한다 — 두 개발자가 다르게 해석할 여지 제거.
- **그룹 옵션은 하드커밋하지 않고 소규모 스파이크로 검증(v3):** 라운드 2 지적 — 스펙 AC1("그룹 엉킴 없이 개별 이동")은 그룹을 **금지하지 않는다**(PowerPoint 단일 레벨 그룹은 더블클릭 진입으로 개별 편집 가능). 또 flat 고정 시 서브시스템(컨테이너+자식 5개) 이동에 매번 6개 수동 다중선택이 필요해 도구가 없애려는 반복 배치 수고를 부분 재도입한다. 따라서 python-pptx에서 불확실한 **커넥터-그룹 상호작용을 실측**한다: *"단일 레벨 그룹(컨테이너+자식) + 최상위 커넥터의 `stCxn`/`endCxn` 바인딩이 그룹 이동 후에도 PowerPoint에서 생존하는가"*. **생존 시** → 그룹을 **플래그 옵션**(`--group-containers`)으로 제공(컨테이너 이동 편의 확보, AC1 공식은 grouped 변형으로 분기). **실패 시** → flat 확정 + "컨테이너 이동 시 자식 비추종·다중선택 필요" caveat를 `docs/review-checklist.md`에 존치. 스파이크 결과를 `_thinking/analysis/00X-container-spike.md`에 기록.
- **왜 계획시점 확정이 아니라 스파이크인가:** 커넥터-그룹 상호작용은 python-pptx의 불확실 영역이라 실측 없는 계획시점 하드커밋은 위험(라운드 2 Architect major·Critic minor). 다만 flat이 AC6-안전한 보수적 기본이라 실패 시 손실은 flat 확정으로 국소 봉합된다.

## 5. Pre-mortem — 6-시나리오 (R1 + 귀속 실패 + 인식 재결합 + 커넥터 라우팅 대비, v2 신설·v3·v4·v5 확장)

R1(재배치 지옥)은 회복 불가능한 1순위 치명 리스크이므로, SHORT 모드라도 정규화 실패 3경로 + **귀속 실패 1경로**(v4 재정의) + **인식 재결합 1경로**(v4 신설) + **커넥터 라우팅 1경로**(v5 신설)를 미리 상정한다.

1. **과교정 (임계값 과대):** 임계값이 넓어 서로 다른 행/열이 하나로 병합·스냅 → 사용자가 일부러 준 오프셋이 소멸 → 그 정렬을 되돌리는 재배치. **조기 신호:** M0 과교정 카운트↑. **방어:** 임계값을 좁게 시작(row_align_tol=박스높이 30%), 정규화 on/off 플래그, M0 실물 실측으로 즉시 포착.
2. **미교정 (임계값 과소):** 임계값이 좁아 정렬돼야 할 박스가 안 붙음 → 줄이 삐뚤·크기 들쭉날쭉 → 사용자가 정렬 재배치(B2가 구조적으로 빠지는 함정). **조기 신호:** M0 미교정 카운트↑. **방어:** M0/M3에서 임계값 단계적 상향, 과교정·미교정 두 카운트의 균형점 탐색.
3. **role 오분류 → 크기통일 오류:** 인식이 role을 잘못 분류하면 다른 역할의 중앙값으로 크기가 통일돼 박스가 엉뚱한 크기가 됨 → 재배치. **조기 신호(v4):** M0 role 서브게이트(role confusion) ↑. **방어:** role 정확도를 M0 인식 서브게이트로 편입(AC3, v4), 크기통일은 같은 role 안에서만 수행, `review-checklist.md`에 role 검수를 필수항목화(AC4), `uniform_box` on/off 플래그.
4. **귀속 실패 — 상류 서브게이트가 하류 AC2 꼬리를 상계 못함(v4 재정의):** 인식 경로 AC2를 게이트로 삼아도, 좌표 서브게이트가 **중앙값**이면 40박스 중 5~8개가 크게 어긋난 fat-tail을 통과시켜 서브게이트가 다 초록인데도 인식 경로 AC2가 fail → 원인 라우팅이 "전부 GO"를 가리켜 **어디를 고칠지 지목 실패**. 또는 elbow 밀집 라우팅 confound가 박스 배치와 무관하게 AC2를 오염. **조기 신호:** 인식 경로 AC2 fail + 모든 서브게이트 GO(잔여 꼬리 케이스). **방어:** 좌표 서브게이트에 **꼬리 지표(p90 centroid·IoU<0.5 노드 ≤10%)** 추가(v4)해 서브게이트가 AC2 꼬리를 실제로 상계, elbow 경로 조악은 AC2 설문에서 별도 배제(§3 설문(3))+M0 실측, 잔여 케이스는 §4 NO-GO 꼬리 경로로 처리.
5. **인식 재결합 → AC7 압박·오손절(v4 신설):** 인식 경로 AC2를 게이트로 삼으면 R1 검증이 인식 성숙도에 재결합 → 인식 좌표가 약하면 인식 경로 AC2가 fail하고, 이를 정규화 실패로 오귀속하면 **멀쩡한 B1을 오손절**하거나, 좌표를 40개 수기이동해 통과시키려다 **AC7(≤10분)을 파괴**. **조기 신호:** 골든 경로 AC2는 GO인데 인식 경로만 fail(= 인식 문제), 또는 AC3 좌표/role 서브게이트 미달. **방어:** 골든/좌표/role 귀속 레이어가 "R1 아님, 인식 문제"를 지목해 오손절 차단(§4 NO-GO), 좌표계 등방 정규화 스키마 선제 방어·좌표 힌트 프롬프트 실효 실측·API vision 승격 예산·**B3를 M2로 앞당겨** AC7 방어(§4 AC2×AC7 해소).
6. **밀집 커넥터 라우팅 조악(R13, v5 신설):** 바인딩 elbow가 40박스 밀집에서 자동 재라우팅으로 박스를 관통·엉킴 → 정렬·크기가 완벽해도 "다시 그리는 게 낫다"의 제2 트리거. B1 vs B2와 독립. **조기 신호:** M0 elbow 게이트에서 박스 관통 >10% 또는 과다 교차 >15%. **방어(§4 M0 elbow 게이트·2d 스파이크):** M0에서 명시 임계로 pass/fail(관통 = 커넥터가 끝점 아닌 노드 bbox 내부 교차, v6 정의), 미달 시 복구 후보(cxnIdx 정책 고정·M1 "고정좌표+저장전 바인딩" 절충)를 **M0 2d 마이크로 스파이크로 저장 경로 생존을 실측한 뒤에만** 확정(v6). AC2 설문(3)로 카운트 격리하되 이 게이트가 책임 판정. **[v6] 스파이크에서 생존 실패 시 유일 대안은 비바인딩 커넥터(AC6 위배)이며 밀집 AC6·AC2 충돌이 환원 불가일 수 있음을 정직히 기록(§6 R13).** R1(회복 불가)과 달리 복구 가능성을 스파이크로 실측하는 성격 차이는 유지.

## 7. Verification Steps (마일스톤별 구체 절차)

**M0 검증 (v6 — M0a 벌거벗은 프로브 → M0b: 좌표계 확정 + 인식 경로 존재적 게이트 + B1/B2 A/B + elbow 게이트/스파이크 + 귀속 레이어, 다표본)**

*— M0a (반나절 프로브·손절 게이트, 아래 0-pre~2a) —*
0-pre. **[v6 착수 전]** `ir/schema.md`·`schema.json`에 좌표계 = **등방 정규화 정수(장축 0~1000, 단축 0~round(1000×단축/장축), canvas는 종횡비만)** 확정, 컨테이너 title 인라인·엣지 라벨 별도 텍스트박스·legend 항목=스와치+라벨 2 shape 렌더 결정 고정. probe/recognition_eval 오차 단위(‰)를 스키마와 일치.
0a. **최소 골든** 2장 라벨링(bbox + 노드/엣지 존재 + role만; intent_align/size_class 없음). **재현성 스팟체크(필수)** 로 라벨 편차 기록(R11).
1a. 각 손그림을 세션이 Read → 인식 IR 작성 → `ir_out/example-0N.ir.json`.
2a. **[M0a 손절 게이트]** `python eval/probe_recognition.py ...` → **centroid·IoU·recall만** 산출 + 좌표 격자·상대위치 힌트 프롬프트 before/after centroid 비교. **좌표 충실도가 가망 없고 힌트 + API vision 승격으로도 못 살리면 여기서 손절**(렌더러·A/B·정규화 미착수라 손실 반나절). 살릴 만하면 M0b(2 이하)로 진입. `00X-m0-slice.md`에 M0a 판정 기록.

*— M0b (M0a 통과 시에만, 아래 0~5) —*
0. `docs/golden-ir-labeling.md` 프로토콜대로 최소 골든에 `intent_align`·`size_class`·**role 정답** 을 더한 전체 골든 IR 완성(**2장, 밴드 진입 시 +1**). **AC2 판정자 캘리브레이션**·라벨 품질 기록(R11).
1. 각 손그림을 세션이 Read → `schema.md`에 맞춰 인식 IR 작성(M0a IR 재사용 가능) → `ir_out/example-0N.ir.json`.
2. **[귀속 서브게이트 계측]** `python eval/recognition_eval.py ir_out/example-0N.ir.json tests/fixtures/example-0N.golden.ir.json` → 재현율·precision·F1 **+ centroid(중앙값·p90)·IoU(중앙값·IoU<0.5 노드 수)·role confusion** 출력. 재현율 ≥0.80 AND precision ≥0.85 AND **중앙값 centroid ≤ 박스폭 X% AND 중앙값 IoU ≥0.7 AND p90 centroid ≤ Y% AND IoU<0.5 노드 ≤ 전체 10% AND role confusion ≤ 임계**를 각각 기록(pass/fail이 아니라 fail 원인 라우팅용). **골든↔인식 diff 크기 출력** + 표본 IR 수정 패스 실소요 측정 → AC7 인식-시간 성분 조기 추정.
2c. **[임계 도출 — 순환 회피]** recognition_eval의 합성 노이즈 스윕으로 골든 좌표에 노이즈를 점증 주입 → 정규화(B1)가 행/열을 오분류하기 시작하는 오차를 측정 → 그 아래로 X·Y·`*_align_tol`을 held-out 도출(§4 M3). **[v6 도출 순서 — 자기참조 회피, 라운드 5 improvement] 붕괴점이 tol에 의존하는 순환을 피하려, 먼저 골든의 참 행/열 구조를 기준으로 정규화가 행/열을 오분류하기 시작하는 붕괴 오차를 측정하고, `*_align_tol`을 그 붕괴 오차 아래로 고정한 뒤, 좌표 허용치 X를 tol보다 더 보수적으로 잡는다(X < tol < 붕괴오차).**
2d. **[elbow 라우팅 게이트 — R13, pass/fail]** 골든 밀집 케이스에서 바인딩 elbow의 초기 렌더 경로·cxnIdx 선택 정책 측정 → **박스 관통 커넥터 ≤ 전체 10%(관통=커넥터가 끝점 아닌 노드 bbox 내부 교차) AND 과다 교차 ≤ 15%** 판정(초기 임계, 여기서 재확정). **[v6 스파이크 — 라운드 5 MAJOR] 이 pptx를 PowerPoint에서 열어 → 아무것도 안 옮긴 상태에서 저장 경로가 재라우팅되는가·초기 경로가 생존하는가·cxnIdx가 재선택되는가를 실측**하고 `00X-m0-slice.md`에 기록. 미달 시 복구 후보 (a) cxnIdx 정책 고정 또는 (b) M1 "고정좌표+저장전 바인딩" 절충은 **이 생존이 실측된 경우에만** 확정, 생존 실패면 비바인딩 커넥터(AC6 위배)가 유일 대안이며 AC6·AC2 밀집 충돌 환원 불가 가능성을 §6 R13에 기록. AC2 설문(3) 카운트 제외 항목의 책임 게이트.
3. **[귀속 기준선]** `normalize/layout.py` + 스텁 렌더러로 **골든 IR→정규화→렌더** pptx 생성 → 의도-기반 과교정·미교정 카운트(합 ≤ 전체 10%, AC2 기계 백업) + PowerPoint 설문 (1)만 카운트한 골든 경로 재배치율 기록(귀속 기준선, pass/fail 아님).
4. **[1순위 존재적 게이트 — 세션 인식 경로 + B1 vs B2 A/B]** 스텁 렌더러(elbow·role 크기통일)로 **세션 인식 IR→정규화→렌더** pptx를 **표본 전부** 생성하되 **정규화 on(B1)·off≈스냅만(B2) 두 벌** → PowerPoint에서 설문 (1)"위치·정렬·크기 틀림"만 카운트(설문 (2) 스타일 조악·(3) 커넥터 경로 조악 제외) / 전체를 **마진 밴드**로 판정(≤7% GO / 7~13% 표본 1장 추가 후 재판정 / >13% NO-GO). **[v5/v6 A/B 판정] B2 대등/우월→B1 폐기·B2 채택, [v6] B1 우월분이 ±3%p 초과 시에만 B1 확정(§8 ADR 실증 연결)·±3%p 이내면 YAGNI로 B2, 둘 다 >13%→좌표 병목 확정·B3 상시화.** **[v6 다표본 집계] 종합 GO/NO-GO는 최악치: 어느 표본이라도 >13%면 NO-GO, 모든 표본 ≤7%면 GO, 그 외 표본 추가 후 재판정(§3 AC2).** **채택 정규화의 인식 경로 AC2 값이 M1 GO/NO-GO를 정한다.** 판정 전 AC2 판정자 캘리브레이션(같은 스텁 2회 판정 카운트 편차 기록).
5. 존재적 게이트 결과 + 귀속 서브게이트 결과를 `_thinking/analysis/00X-m0-slice.md`에 기록하고 GO/NO-GO 판단. **인식 경로 NO-GO 시 §4 NO-GO 라우팅으로 원인 진단**(골든도 fail→정규화, 골든 GO+좌표/role fail→인식, 전부 GO인데 fail→꼬리 케이스 경로). 밴드 진입 시 표본 추가 후 재판정.

## 8. ADR (Architecture Decision Record)

- **Decision:** IR(JSON)을 인식·검수·렌더의 단일 계약으로 삼아 **확률적 세션 인식**과 **결정론적 파이썬 렌더러**를 분리하고, **IR 좌표계는 등방(isotropic) 정규화 정수(장축 0~1000, 단축 비례; v6 — v5의 축별 독립 0~1000이 남긴 이방성 척도 구멍 교정, R10 스키마 선제 방어)**, 레이아웃 정규화를 **클러스터링(B1) 기반 품질 1순위**로 두되 **B1 채택은 M0에서 B2(스냅만) 대비 실제 세션 인식 좌표 위 A/B 실증으로 확정(v5) — 단 B1 우월분이 밴드 노이즈(±3%p)를 초과할 때만 채택, 이내면 YAGNI로 B2(v6)** 한다. **컨테이너 기본은 평면 배경 사각형(비그룹, title 인라인)**, 그룹 옵션은 M1 스파이크로 검증 후 플래그 제공. 존재적 리스크(R1)를 **M0 얇은 수직 슬라이스에서 실물 pptx로 최초 검증하되, pass/fail은 사용자가 실제 겪는 세션 인식 경로 AC2로 판정(v4)** 하고 — **[v6] M0를 M0a(반나절 벌거벗은 인식-좌표 프로브 = 하중 전제 R10 단독·선행 손절 게이트)와 M0b(통과 시에만 렌더러·A/B·게이트 건설)로 분리해 값싼 조기 손절 규율을 복원** — **골든 IR 경로·좌표·role 서브게이트는 그 자체가 게이트가 아니라 fail 원인을 정규화 vs 인식으로 라우팅하는 귀속(attribution) 레이어**로 둔다. 좌표 서브게이트는 **중앙값 + 꼬리 지표(p90·IoU<0.5)** 로 AC2 꼬리를 상계하고, **role 정확도를 M0 인식 서브게이트로 편입**한다. **밀집 커넥터 라우팅 조악(R13)은 M0 명시 게이트(박스 관통 ≤10%·과다 교차 ≤15%, 관통=끝점 아닌 노드 bbox 내부 교차)로 판정하고, 미달 시 복구 후보(cxnIdx 고정/M1 절충)는 M0 2d 스파이크로 저장 경로 생존을 실측한 뒤에만 확정하며 생존 실패 시 AC6·AC2 밀집 충돌이 환원 불가일 수 있음을 정직히 기록(v6)**. **B3(정렬 힌트)는 잔여 위치-교정 채널로 M2 옵션 전진(v5)**. 출력은 네이티브 도형 + 바인딩 커넥터, 저장은 비파괴 증분.
- **Drivers:** ① 재배치 최소화(R1 치명 실패 방지, AC2 1순위) ② 완전한 편집성 ③ 최소 인프라·저비용 반복 검수.
- **Alternatives considered:** A2(세션이 python-pptx 직접 생성 — IR 없음), **B2(정규화 최소·그리드 스냅만 — v5: 계획시점 하드 기각 철회, M0 A/B의 대조군으로 승격. B1이 실측으로 못 이기면 B2 채택=YAGNI)**, **B3(반자동 정렬 힌트 — R1에 강하나 자동 배치 대체 시 전제 실패이므로 잔여 교정용으로 한정, v5: M2 옵션 전진)**, **IR 좌표계 절대px(v1 초안) — VLM 좌표 안정성이 정규화보다 낮아 기각, 정규화 채택(v5)**. **IR 좌표계 축별 독립 정규화 0~1000(v5안) — 비정방 캔버스에서 1‰(x)≠1‰(y)로 centroid 2D 거리·임계·클러스터링이 척도 혼합 → v6 기각, 등방(장축 0~1000·단축 비례) 채택(축별 임계 분리도 가능하나 등방이 거리 연산 전부를 단일 척도로 통일)**. **컨테이너 그룹핑(단일 레벨) — AC1 위배는 아니나(더블클릭 개별 편집) 커넥터-그룹 바인딩 불확실 → M1 스파이크로 판정 보류.** **elbow 초기경로 완전제어 위해 커넥터 비바인딩 렌더 — AC6 위배(박스 이동 시 미추종)라 기각, 대신 M1 "고정좌표 렌더 + 저장전 바인딩" 절충으로 AC6·AC2 양립(v5).** 스펙에서 이미 기각된 graphviz·SVG 경유·이미지 출력·OpenCV 인식·검수용 오버레이/HTML은 재론하지 않음.
- **Why chosen:** IR 경유(A1)만이 인식 오류 국소 수정·스타일 재현성·골든 테스트를 동시에 확보하고 검수 루프를 값싸게 만든다. 클러스터링(B1)이 과교정·미교정을 **의도(intent_align) 기준** 하나로 함께 조율해 R1을 정면 해소할 **잠재력**이 가장 크다 — **단 [v5] 그 우월성은 원리론이 아니라 M0에서 B2 대비 실제 세션 인식 좌표 위 A/B 실측으로 확정**한다(좌표가 정확하면 B2로 충분·B1은 과잉설계, 부정확하면 B1이 붕괴하는 밴드 의존성 때문). B2가 이기면 단순한 B2를 택하는 것이 YAGNI 원칙에 맞다. **등방 정규화 좌표계(장축 0~1000, 단축 비례)는 VLM 좌표 안정성 때문에 절대px보다 우월**하고 R10을 스키마 설계로 선제 방어하며, 축별 이방성 척도 혼합도 제거한다(v6). **컨테이너 평면은 AC1 카운팅과 AC6 바인딩을 안전하게 지키는 보수적 기본**이되, 그룹 편의는 M1 스파이크로 실측해 결정한다(라운드 2). **[v4 핵심] R1 존재적 pass/fail을 사용자가 실제 겪는 세션 인식 경로로 판정하고 골든/좌표/role을 귀속 레이어로 재배치한 것**은, v3가 골든(사용자가 결코 살지 않는 반사실) 경로로만 게이트해 성공을 반사실 위에서 재던 결함(라운드 3)을 교정하기 위함이다 — 귀속 레이어는 v3 분해의 진짜 가치("못생기면 R1이냐 R10이냐")를 보존한 채 게이트를 현실에 재결합한다. 꼬리 지표·role M0 편입은 상류 서브게이트가 하류 AC2 꼬리를 실제로 상계하게 만든다.
- **Consequences:** (긍정) 인식/렌더 독립 진화, config 교체만으로 스타일 전환, 회귀 테스트 가능, **존재적 리스크를 사용자가 실제 겪는 경로로 M0에서 조기·정직하게 확인**(반사실 위 명목 승리 제거), 좌표 오차·role 오분류를 정규화 붕괴 전에 조기 포착, 상류 게이트가 AC2 꼬리를 상계. (부정) **[v4 정직한 비용] 인식 경로 게이트는 R1 검증을 인식 성숙도에 재결합**한다 — 인식 좌표/role이 약하면 인식 경로 AC2가 fail해 M0에서 인식 개선 서브루프(프롬프트/좌표 힌트/API vision 승격)가 필요할 수 있고, 이는 "값싼 M0"의 서사를 **부분 되감는다**(M0 소요 1~2일→2.5~3일 = M0a 반나절+M0b 2~2.5일, v6.1 표기 정정). 그러나 M0의 본질 가치는 '값쌈'이 아니라 **회복 불가 R1을 풀 렌더러(M3, 6~8일 뒤) 완성 전에 실물로 검증**함이며, 그 가치는 재결합과 무관하게 보존되고 M3 발견 대비 압도적으로 싸다. 또 귀속 레이어가 "R1 아님, 인식 문제"를 지목해 멀쩡한 정규화 접근의 오손절을 막는다. **AC2×AC7 조인트 텐션**(약한 좌표→40 bbox 수기이동→AC7 위협)은 좌표 힌트 실효 실측·API vision 예산·B3 승격으로 해소(§4). **[v5 추가] B1 채택을 실증에 재결합**하면 M0에서 B1이 폐기되고 B2로 단순화될 수 있으나, 이는 손실이 아니라 YAGNI 실현(불필요한 클러스터링 유지비 회피)이다. **커넥터 라우팅 게이트(R13)** 는 정렬·크기와 독립된 제2 실패 경로를 M0에서 책임지고 잡되, 복구 후보(cxnIdx/M1 절충)는 M0 2d 스파이크로 저장 경로 생존을 실측한 뒤에만 확정되며(v6) 생존 실패 시 비바인딩 커넥터가 유일 대안이라 프로젝트 손절로 번지진 않으나 AC6·AC2 밀집 충돌 완전 해소는 보장되지 않는다. **좌표계 정규화 확정**은 R10을 스키마 단계에서 선제 방어하나 세션 프롬프트·렌더러가 0~1000 단위를 일관 준수해야 하는 계약 부담을 추가한다(검증기로 강제). **B3 M2 전진**은 잔여 위치오차의 값싼 채팅 교정 채널을 확보해 AC7을 방어하나 자동 배치 대체로 오용되면 전제 실패 신호이므로 잔여 교정용으로만 위치. **[v6] M0a/M0b 분리**는 최상류·최불확실 하중 전제(R10 인식 좌표 충실도)를 스텁 렌더러·A/B 건설 전에 홀로·먼저 반나절 프로브로 게이트해, v5가 "값싼 M0 부분 되감김"으로 수용하던 리스크-우선순위 역전을 **되감김의 최소화**로 전환한다(거의 무료, 현행 M0를 strictly dominate). **[v6] 등방 좌표계 확정**은 centroid 2D 거리·클러스터링·임계가 단일 척도에서 돌게 해 이방성 왜곡을 제거하나, 세션 프롬프트·렌더러·recognition_eval이 "단축은 0~1000이 아니라 비례 상한(예: 562)"이라는 등방 규약을 일관 준수해야 하는 계약 부담을 추가한다(검증기로 강제). **[v6] R13 복구책의 스파이크 강등**은 미검증 낙관을 실측으로 대체하되, 저장 경로 생존이 실패하면 밀집에서 AC6(바인딩)과 AC2(초기 경로 정결)가 환원 불가로 충돌할 수 있음을 정직히 노출한다(계획이 이를 은폐하지 않고 §6 R13에 기록).
